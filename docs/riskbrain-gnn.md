# RiskBrain GNN 风控实验模块

`ml/riskbrain_gnn/` 用于离线探索银行交易风险评分，包括数据注入、图特征构建、树模型基线、GNN 模型训练和批量评分接入。

## 定位

该模块不是实时交易拦截服务，而是离线算法实验与演示链路：

1. 从公开数据集或本地仿真数据构建交易图。
2. 训练树模型或 GNN 模型，得到风险概率。
3. 将评分结果批量写入系统数据库。
4. 在后台风控、客户提醒、图谱看板中展示解释结果。

## 数据策略

大型数据集不进入 Git 仓库。建议目录：

```text
ml/riskbrain_gnn/data/       # 原始或清洗后的本地数据
ml/riskbrain_gnn/outputs/    # 特征、评分、训练日志
ml/riskbrain_gnn/models/     # 模型权重
```

以上目录已被 `.gitignore` 忽略。

## 标签泄露约束

训练、验证、测试划分应优先按时间或实体冷启动方式执行，避免同一笔交易、同一强关联实体或未来统计特征泄露到训练集。

建议保留三类评估：

- 时间外推：用早期交易训练，用后期交易测试。
- 实体冷启动：验证新账户、新收款方场景。
- 高召回风控：重点观察 fraud recall、PR-AUC、top-k 命中率和人工审核量。

## 与主系统集成

主系统只依赖评分结果，不依赖训练环境：

- 训练和推理可以在 GPU 服务器执行。
- iBank Web 只读取已落库的风险分数、原因码和图谱摘要。
- 远程服务器关闭时，主系统仍可运行，只是不会刷新新模型评分。
